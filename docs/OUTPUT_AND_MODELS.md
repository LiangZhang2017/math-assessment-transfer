# Output folder and supported models

## Output layout

All run outputs go under **`output/`**, one subfolder per model, then per task (**solving**, **assessment**):

```
output/
  gpt-4/
    solving/          # Task 1: problem-solving
      gsm8k-0.json
      ...
      summary.txt
    assessment/       # Task 2: step-level assessment
      gsm8k-0.json
      ...
      summary.txt
  gpt-4o/
    solving/
    assessment/
  o1/
    ...
```

Each **`<id>.json`** (e.g. `gsm8k-0.json`) contains:
- **id**, **problem**
- **llm_solution**: full step-by-step solution text generated by the LLM
- **llm_answer**: final answer extracted from the LLM-generated solution
- **gold**, **correctness**

**`summary.txt`**: one line, e.g. `Correct: 380 / 400 = 95.00%`.

You can run different models and compare by inspecting `output/<model>/solving/*.json` or `output/<model>/assessment/*.json` and their `correctness` / `correctness_llm_label` fields.

## Models (Azure deployment names)

Use the **exact deployment name** as in your Azure OpenAI resource:

| Model type   | Typical deployment name | Notes |
|-------------|-------------------------|--------|
| GPT-4        | `gpt-4`                 | Default in this repo. |
| GPT-4o      | `gpt-4o`                | Often faster/cheaper. |
| GPT-4o mini | `gpt-4o-mini`           | Lighter. |
| Reasoning   | `o1`, `o1-mini`, `o3`, `o3-mini`, `o4-mini` | If available in your region. |
| GPT-5       | e.g. `gpt-5` or as in Azure catalog | Depends on your Azure offering. |

- Set **`AZURE_OPENAI_MODEL`** in `.env` for the default, or pass **`--model <name>`** to `scripts/run_problem_solving_eval.py`.
- Names must match what you created in Azure (e.g. in Azure AI Studio → Deployments). If a model returns 404, check the deployment name and region/availability.

## Commands

```bash
# Default model from .env (e.g. gpt-4) → output/gpt-4/solving/
python scripts/run_problem_solving_eval.py

# Specific model → output/gpt-4o/solving/
python scripts/run_problem_solving_eval.py --model gpt-4o

# Assessment task → output/gpt-4o/assessment/
python scripts/run_assessment_eval.py --model gpt-4o

# First N items only (for testing)
python scripts/run_problem_solving_eval.py --model gpt-4o --limit 20
```

- **Solving**: output directory is **`output/<model>/solving/`**; per-id JSON, `summary.txt`, and (if using run_solve_only) `responses.json` are written there.
- **Assessment**: output directory is **`output/<model>/assessment/`**; per-id JSON and `summary.txt` are written there.
